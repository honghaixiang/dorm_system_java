# 学生宿舍管理系统 - 后端更新说明

## 🎉 本次更新（前后端分离版本）

### ✅ 已添加的配置

1. **CORS跨域配置** (`CorsConfig.java`)
   - 支持前后端分离
   - 允许携带Cookie凭证
   - 支持跨域请求

2. **Session持久化配置** (`SessionConfig.java`)
   - 解决Session丢失问题
   - 配置Cookie序列化
   - 支持跨域Session

3. **Cookie安全配置** (`application.properties`)
   - 配置Cookie属性
   - 设置Session超时
   - 优化安全性

### 📂 新增文件

```
src/main/java/com/dorm/config/
├── CorsConfig.java       # CORS跨域配置
└── SessionConfig.java    # Session配置
```

### 🚀 使用方法

#### 1. 启动后端
```bash
# Windows
start.bat

# Linux/Mac
./start.sh
```

后端将运行在：`http://localhost:8080`

#### 2. 测试账号
- **管理员账号**：
  - 用户名：`admin`
  - 密码：`admin123`

- **学生账号**：
  - 通过注册页面创建

#### 3. 前端连接
确保前端的 `API_BASE_URL` 配置为：
```javascript
const API_BASE_URL = 'http://localhost:8080/api';
```

### 🔧 配置说明

#### CORS配置
- 允许所有源（开发环境）
- 支持凭证传递
- 允许所有HTTP方法

#### Session配置
- 超时时间：30分钟
- Cookie名称：JSESSIONID
- HttpOnly：启用
- SameSite：Lax

### ⚠️ 注意事项

1. **首次启动**：
   - 会自动创建数据库
   - 会自动插入admin账号
   - 数据库文件位于 `./data/` 目录

2. **重置数据库**：
   - 停止服务
   - 删除 `./data/` 目录
   - 重新启动服务

3. **端口占用**：
   - 确保8080端口未被占用
   - 可在 `application.properties` 中修改端口

4. **H2控制台**：
   - 访问：`http://localhost:8080/h2-console`
   - JDBC URL：`jdbc:h2:file:./data/dorm;AUTO_SERVER=TRUE`
   - 用户名：`sa`
   - 密码：（留空）

### 🐛 常见问题

#### 问题1：登录后自动跳转回登录页
✅ 已通过本次更新解决

#### 问题2：CORS错误
✅ 已添加CORS配置解决

#### 问题3：数据库初始化失败
**解决方案**：
1. 删除 `./data/` 目录
2. 重启服务

#### 问题4：无法访问H2控制台
**解决方案**：
确保在 `application.properties` 中：
```properties
spring.h2.console.enabled=true
```

### 📊 数据库表结构

- `sys_user` - 用户表
- `dorm` - 宿舍表
- `assignment` - 宿舍分配表
- `repair` - 报修表
- `fee` - 费用表
- `violation` - 违规表

### 🔗 API接口

所有API接口前缀：`/api`

#### 认证相关
- `POST /api/register` - 注册
- `POST /api/login` - 登录
- `POST /api/logout` - 登出

#### 用户相关
- `GET /api/profile` - 获取个人信息
- `PUT /api/profile` - 更新个人信息
- `GET /api/students` - 获取学生列表

#### 宿舍相关
- `GET /api/dorms` - 获取宿舍列表
- `POST /api/dorms` - 添加宿舍

#### 其他功能
详见API文档
