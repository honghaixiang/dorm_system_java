## 🎯 架构说明

### 整合前（前后端分离）
```
前端: localhost:8000 (npm run serve)
  ↓ 跨域请求
后端: localhost:8080 (Spring Boot)
  ↓ Cookie 无法发送
问题: Session 丢失 ❌
```

### 整合后（前后端统一）
```
访问: localhost:8080/login.html
  ↓ 加载前端页面
前端 JavaScript 调用: localhost:8080/api/login
  ↓ 同域请求，Cookie 自动发送
后端: 识别 Session，登录成功 ✅
```

## 💡 常见问题

### Q1: 访问 localhost:8080 显示 404？
**A:** Spring Boot 需要访问具体文件，访问 `http://localhost:8080/login.html`

### Q2: 页面样式丢失？
**A:** 确保 `style.css` 在 `static` 文件夹中（已包含）

### Q3: 登录后还是跳回登录页？
**A:** 这应该不会发生了！因为前后端同域，Cookie 会自动发送。如果还有问题，请检查：
- 后端服务是否正常启动
- 浏览器控制台是否有错误
- 清除浏览器缓存后重试

### Q4: 想查看数据库数据？
**A:** 访问 H2 数据库控制台：
```
http://localhost:8080/h2-console
JDBC URL: jdbc:h2:file:./data/dorm
用户名: sa
密码: (留空)
```
